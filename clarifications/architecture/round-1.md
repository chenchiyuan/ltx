# 架构方向 - Round 1

## 大原则确认

### 目标
基于 ComfyUI 和 LTX 2.3 搭建一套对外提供文生视频、图生视频服务，对内管理工作流和 GPU 任务的系统服务。

### 范围
- 包含: 对外友好界面、文生视频、图生视频、内部工作流管理、LTX 2.3 GPU 部署、GPU 服务发现、任务重试、任务计数。
- 不包含: 具体代码实现、正式 architecture.md、云厂商采购方案、模型训练方案。

---

### CLR-ARCH-001: 基础能力范围
- **模糊点**: 系统第一阶段需要覆盖哪些视频生成能力。
- **影响范围**: 产品入口、工作流模板、任务队列和 GPU Worker 能力边界。
- **推荐选项**: 待后续架构设计阶段确认。
- **结论**: 第一阶段需要对外提供文生视频和图生视频服务，并提供友好的用户界面。
- **来源分类**: user_confirmed
- **状态**: 生效
- **确认时间**: 2026-07-13

### CLR-ARCH-002: 内部管理范围
- **模糊点**: 内部后台需要管理哪些生产对象。

- **推荐选项**: 待后续架构设计阶段确认。
- **结论**: 内部需要管理文生视频、图生视频工作流，并支持任务系统、任务重试和任务计数。
- **来源分类**: user_confirmed
- **状态**: 生效
- **确认时间**: 2026-07-13

- **模糊点**: GPU 集群初始资源边界。
